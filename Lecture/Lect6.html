Forms Overview

Web forms are interactive elements on web pages that allow users to input data, such as text, choices, or selections, and submit it for processing.

Parts of a Web Form

A form contains controls (widgets) that allow user interaction:

  • Controls: Input boxes, radio buttons, selection lists, checkboxes, text areas
  • Widgets: Spin boxes, slider controls, calendar controls, color pickers

The <form> Tag

The form tag defines a form and includes several attributes:

Important Attributes: action, method, target, enctype

Basic Form Example

The action Attribute

Specifies the URL that processes the form data:

<form action="/submit-form.php">

The method Attribute

Sets how form data is sent to the server:

  • GET: Appends data to URL (visible, limited length)
  • POST: Sends data in HTTP request body (not visible, no size limit)

Creating Effective Forms

Best Practices:
  • Make forms assist users as much as possible
  • Adjust field sizes to indicate expected input
  • Use appropriate input types for each data field
  • Structure forms logically with clear sections
  • Avoid wide input elements that require horizontal scrolling
  • Maintain consistent design patterns

User Interface Considerations

  • Learnability: Intuitive for new users
  • Efficiency: Quick completion of tasks
  • Memorability: Easy recall after time away
  • Error Tolerance: Easy recovery from mistakes
  • Satisfaction: Pleasant user experience
