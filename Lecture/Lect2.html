Website Preparation

Before You Start Creating Websites

You must first ask these questions:

  • What are the tools the professionals use?
  • What will my website look like?
  • What are the important files that I should have?

What Professionals Use

  • A Computer
  • Text Editor, to write the code in
  • Web Browsers, to test the code in
  • Graphic Editing Software (Gimp, Figma, Photoshop)
  • Version Control System (GitHub, GitLab)
  • FTP program (optional, as Git is increasingly replacing FTP)
  • Automation System to automatically perform repetitive tasks
  • Libraries, frameworks, etc.

What You Need to Prepare

  • Computer - Any specs will do, but for serious web development, invest in your computer
  • Text Editor - Default Notepad, or start with Notepad++
  • Web Browser - Chrome, Firefox, Opera, Microsoft Edge, Safari, Brave

What Will My Website Look Like?

Web Design Principle: Plan Your Site Before You Build It

  • Clearly define your site's purpose

The basic structure of your website FOLDER should have at least:

HTML5

After HTML 4.01 was released, focus shifted to XHTML and its stricter standards. XHTML 2.0 had even stricter standards than 1.0, rejecting web pages that did not comply. It fell out of favor gradually and was abandoned completely in 2009.

HTML5 is much more tolerant and can handle markup from all the prior versions. Though HTML5 was published officially in 2012, it has been in development since 2004.

What is HTML?

  • HTML (Hypertext Markup Language) is not a programming language
  • It is a markup language that tells web browsers how to structure web pages
  • It can be as complicated or as simple as the web developer wants it to be

What is HTML5?

  • HTML5 is the newest version of HTML
  • It incorporates all features from earlier versions of HTML, including the stricter XHTML
  • It adds a diverse set of new tools for the web developer to use
  • It is still a work in progress

Goals of HTML5

  • Support all existing web pages (no requirement to revise older websites)
  • Reduce the need for external plugins and scripts
  • Improve the semantic definition of page elements
  • Make the rendering of web content universal and device-independent
  • Handle web document errors in a better and more consistent fashion

First Look at HTML5

XHTML DOCTYPE declaration:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

HTML5 DOCTYPE declaration:

<!DOCTYPE html>

NOTE: The DOCTYPE tells the browser which type and version of document to expect. This should be the last time the DOCTYPE is ever changed. From now on, all future versions of HTML will use this same simplified declaration.

XHTML <html> element:

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en">

HTML5 <html> element:

<html lang="en">

NOTE: The lang attribute declares which language the page content is in. Though not strictly required, it should always be specified, as it can assist search engines and screen readers.

Basic HTML Structure

Basic HTML5 Web Page

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>My First HTML5 Page</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <p>HTML5 is fun!</p>
</body>
</html>

HTML Page Structure Explained

  • <!DOCTYPE html> - Defines that this document is an HTML5 document
  • <html></html> - Wraps all content on the page (root element)
  • <head></head> - Container for non-content elements (metadata, links, etc.)
  • <meta charset="utf-8"> - Specifies the character set (UTF-8 includes most characters)
  • <title></title> - Sets the page title (appears in browser tab)
  • <body></body> - Contains all visible content (text, images, videos, etc.)

Four Key Concepts

  1. Elements - All HTML pages are made up of elements (containers for page content).
    Example: <h1>Text Here</h1>
  2. Tags - HTML codes enclosed in angle brackets used to format text.
    Example: <p>

    An element is made up of two tags: an opening tag and a closing tag.

  3. Attributes and Values - Define characteristics of an element.
    Example: <p align="center">Centered text</p>

    Not all attributes and values work together with all elements. Always enclose values in quotation marks.

  4. Nesting - Elements must never overlap.
    Example of proper nesting: <p><strong>Text</strong></p>
    Example of improper nesting: <p><strong>Text</p></strong>

HTML Is Not Case Sensitive

  • HTML tags are not case sensitive: <P> means the same as <p>
  • The HTML5 standard does not require lowercase tags, but W3C recommends lowercase

HTML Comments

<!-- Add your Comment Here -->

Comments are not displayed in the browser; they are sometimes called non-printing lines.

Semantic Elements

What are Semantic Elements?

  • Semantics is the study of the meanings of words and phrases
  • Semantic elements = elements with a meaning
  • They clearly describe their meaning to both the browser and the developer

Semantic vs Non-semantic Elements

Non-semantic Elements Semantic Elements
<div> and <span> <form>, <table>, <article>
Tells nothing about its content Clearly defines its content

HTML5 Semantic Elements

<section> Element

Defines a section in a document (thematic grouping of content).

<section>
  <h1>WWF</h1>
  <p>The World Wide Fund for Nature (WWF) is...</p>
</section>

<article> Element

Specifies independent, self-contained content (forum post, blog post, news article).

<article>
  <h1>What Does WWF Do?</h1>
  <p>WWF's mission is to stop the degradation of our planet's...</p>
</article>

<header> Element

Specifies a header for a document or section (introductory content).

<article>
  <header>
    <h1>What is a Book?</h1>
    <p>Book definition</p>
  </header>
  <p>A book is...</p>
</article>

<footer> Element

Specifies a footer for a document or section (author, copyright, contact info).

<footer>
  <p>Posted by: Hege Refsnes</p>
  <p>Contact: <a href="mailto:someone@example.com">someone@example.com</a></p>
</footer>

<nav> Element

Defines a set of navigation links (for major blocks only).

<nav>
  <a href="/html/">HTML</a>
  <a href="/css/">CSS</a>
  <a href="/js/">JavaScript</a>
</nav>

<aside> Element

Defines content aside from the main content (like a sidebar).

<p>What is a Book?</p>
<aside>
  <h4>Book</h4>
  <p>A book is a medium for...</p>
</aside>

<figure> and <figcaption> Elements

Groups an image with its caption.

<figure>
  <img src="panda.png" alt="Panda">
  <figcaption>Fig1. – This is Panda</figcaption>
</figure>

Text Formatting

Saving and Viewing Pages

  • Save files with .html extension (.htm or .html both work)
  • Your web page will take the icon of your default browser

Headings

The purpose of heading elements is to indicate different heading levels:

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>

h1 is the largest and h6 is the smallest.

Horizontal Rule

The <hr> element draws a horizontal line to separate content:

<hr>

Text Flow

Control text flow with these elements:

  • <br> - Line break (single line break)
  • <p> - Paragraph (adds blank line before new paragraph)

Indenting Text

  • Use non-breaking space entities for small indents: &nbsp;
  • Use <blockquote> for large text blocks
  • Use <pre> to retain text formatting (as typed)

Text Alignment

Use the align attribute to position text:

<p align="right">Right-aligned text</p>
<p align="center">Centered text</p>
<p align="justify">Justified text</p>

NOTE: The browser usually defaults to left alignment. Always check how your pages display.
